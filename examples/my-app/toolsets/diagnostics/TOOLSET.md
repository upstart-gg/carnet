# Diagnostics & Quality Assurance Tools

Tools for validating code, debugging errors, and running quality checks.

## Overview

These tools help you catch errors early, understand what went wrong, and ensure code quality before delivery.

## Tool Categories

### Code Validation
- `lint` - Check code quality and style

### Error Diagnosis
- `diagnoseClientError` - Understand client-side errors

### Quality Assurance
- `runQA` - Run comprehensive QA checks

## Best Practices

- Run `lint` after making code changes
- Use `diagnoseClientError` when users report issues
- Run `runQA` before reporting task completion
- Fix all lint errors before delivery
